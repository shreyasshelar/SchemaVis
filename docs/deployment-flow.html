Deployment Flow

Press Play to animate what happens from a git push to live pods updating.

1
📝
git push origin main
You push code to the main branch on GitHub. This is the only manual action in the entire pipeline.
GitHub
2
⚙️
GitHub Actions — CI starts
The ci.yml workflow triggers on push to main. Compiles the Java backend, type-checks the TypeScript frontend, runs the build. Fails fast if anything breaks.
GitHub Actions
3
🐳
Docker images built + pushed
The deploy.yml workflow builds two multi-stage Docker images (backend + frontend) using BuildKit layer caching. Tags: sha-abc1234 and latest. Pushed to ghcr.io (private GitHub Container Registry).
ghcr.io
4
🔍
ArgoCD Image Updater detects new tag
Image Updater polls ghcr.io every 2 minutes. It sees the new sha-abc1234 tag matching the ^sha- pattern. It commits updated image tags to the git repo — creating an audit trail entry.
ArgoCD Image Updater
5
🔄
ArgoCD detects git change → diffs cluster
ArgoCD sees the new commit (image tag update). It computes a diff between the desired state (git) and the live cluster. Only the backend and frontend Deployments have changed — everything else is untouched.
ArgoCD
6
🚀
Rolling update — zero downtime
Kubernetes starts new pods with the new image before terminating old ones (RollingUpdate strategy). Health checks (Spring Actuator /actuator/health/readiness) must pass before traffic is shifted. HPA adjusts replicas based on load.
k3s
7
☁️
Cloudflare tunnel serves live traffic
The cloudflared pods (already running, untouched by this deploy) forward requests from schemavis.shreyasshelar.uk to the new frontend pods. PostgreSQL and cloudflared were never restarted.
Cloudflare
8
✅
Deploy complete — audit trail in git
The git history now has a commit from ArgoCD Image Updater: "chore: update backend to sha-abc1234". Roll back at any time by reverting that commit — ArgoCD self-heals within seconds.
GitOps

✓ Deployment Complete

Total time from git push to live: ~4–6 minutes (dominated by Maven build + Docker layer cache warm-up).
Subsequent deploys with warm cache: ~2–3 minutes.
