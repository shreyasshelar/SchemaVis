② k3s (Kubernetes)
✓
Install k3s
Single command — includes kubectl, Helm-ready, metrics-server
▾
curl -sfL https://get.k3s.io | sh - # Copy kubeconfig so kubectl works as your user mkdir -p ~/.kube sudo cp /etc/rancher/k3s/k3s.yaml ~/.kube/config sudo chown $USER:$USER ~/.kube/config # Verify kubectl get nodes
✓
Install Helm
Package manager for Kubernetes
▾
curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash helm version
③ ArgoCD + Image Updater
✓
Install ArgoCD
GitOps engine — reconciles git → cluster
▾
kubectl create namespace argocd kubectl apply -n argocd \ -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml # Wait for ArgoCD to be ready kubectl wait --for=condition=available deployment/argocd-server \ -n argocd --timeout=120s # Get initial admin password kubectl -n argocd get secret argocd-initial-admin-secret \ -o jsonpath="{.data.password}" | base64 -d && echo
✓
Install ArgoCD Image Updater
Watches ghcr.io and commits new image tags to git
▾
kubectl apply -n argocd \ -f https://raw.githubusercontent.com/argoproj-labs/argocd-image-updater/stable/manifests/install.yaml
✓
Create ghcr.io pull secret
Lets Image Updater read private image tags
▾
Create a GitHub PAT with read:packages scope at github.com → Settings → Developer settings → PAT.
# Replace YOUR_GITHUB_TOKEN with your PAT kubectl create secret docker-registry ghcr-credentials \ --namespace argocd \ --docker-server=ghcr.io \ --docker-username=shreyasshelar \ --docker-password=YOUR_GITHUB_TOKEN
✓
Create git write-back secret
Lets Image Updater commit new image tags to your repo
▾
Create a GitHub PAT with repo (write) scope — separate from the read-only one above.
# Replace YOUR_WRITE_TOKEN kubectl create secret generic git-write-token \ --namespace argocd \ --from-literal=password=YOUR_WRITE_TOKEN \ --from-literal=username=shreyasshelar
④ Application Secrets (one-time, on VM)
✓
Create namespace + backend secret
DB password, JWT key, AI API keys
▾
kubectl create namespace schema-vis kubectl create secret generic schema-vis-backend-secret \ --namespace schema-vis \ --from-literal=SPRING_DATASOURCE_PASSWORD='your-db-password' \ --from-literal=JWT_SECRET='your-64-char-jwt-secret-here-make-it-long!!' \ --from-literal=GEMINI_API_KEY='your-gemini-key' \ --from-literal=GROQ_API_KEY='your-groq-key' \ --from-literal=OPENROUTER_API_KEY='your-openrouter-key'
✓
Create postgres + cloudflared secrets
DB credentials and Cloudflare tunnel token
▾
kubectl create secret generic schema-vis-postgres-secret \ --namespace schema-vis \ --from-literal=POSTGRES_PASSWORD='your-db-password' \ --from-literal=POSTGRES_USER=schemavis \ --from-literal=POSTGRES_DB=schemavis kubectl create secret generic schema-vis-cloudflared-secret \ --namespace schema-vis \ --from-literal=TUNNEL_TOKEN='your-cloudflare-tunnel-token'
⑤ Cloudflare Tunnel
✓
Create tunnel in Cloudflare Zero Trust
one.dash.cloudflare.com → Networks → Tunnels
▾
1. Go to one.dash.cloudflare.com
2. Networks → Tunnels → Create a tunnel
3. Name: schema-vis
4. Copy the tunnel token → use it in the secret above
5. Public Hostname tab → Add hostname:
  Subdomain: schema · Domain: shreyasshelar.uk
  Service: http://schema-vis-frontend.schema-vis.svc.cluster.local:80
DNS CNAME is created automatically by Cloudflare — no manual DNS entry needed.
✓
Add argocd.shreyasshelar.uk hostname
Expose ArgoCD UI via the same tunnel
▾
In the same tunnel, add a second Public Hostname:
Subdomain: argocd · Domain: shreyasshelar.uk
Service: http://argocd-server.argocd.svc.cluster.local:80
⑥ First Deploy
✓
Apply the ArgoCD Application
This is the only helm/kubectl command you'll ever run for deploys
▾
# Clone your repo first (or git pull) git clone https://github.com/shreyasshelar/SchemaVis.git cd SchemaVis kubectl apply -f k8s/argocd-app.yaml # Watch ArgoCD sync kubectl get application schema-vis -n argocd -w
ArgoCD will pull the Helm chart from GitHub and deploy all services. Takes ~2–3 minutes.
