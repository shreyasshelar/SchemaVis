GCP VM — create & SSH in
▼
Create e2-medium VM in asia-south1
GCP Console → Compute Engine → Create instance.
Machine: e2-medium (2 vCPU, 4 GB) · Region: asia-south1 (Mumbai) · OS: Ubuntu 22.04 LTS · Disk: 30 GB.
Tick "Allow HTTP/HTTPS traffic" (optional, for reference).
SSH into the VM
Click the SSH button in the GCP console or use gcloud compute ssh schemavis-vm --zone=asia-south1-a
Update & install tools
sudo apt update && sudo apt upgrade -y
sudo apt install -y curl wget git nano openssl
2
Install k3s + Helm
▼
Install k3s (Traefik disabled)
We disable Traefik — Cloudflare Tunnel handles all inbound traffic.
curl -sfL https://get.k3s.io | sh -s - --disable traefik

# Wait 20 seconds, then verify:
sudo k3s kubectl get nodes
Set up kubectl for your user
mkdir -p ~/.kube
sudo cp /etc/rancher/k3s/k3s.yaml ~/.kube/config
sudo chown $USER:$USER ~/.kube/config
kubectl get nodes  # should work without sudo
Install Helm 3
curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash
helm version
3
Install ArgoCD + Image Updater + CLI
▼
Install ArgoCD
kubectl create namespace argocd
kubectl apply -n argocd \
  -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml

# Wait for pods (takes ~2 min)
kubectl wait --for=condition=Ready pod \
  -l app.kubernetes.io/name=argocd-server -n argocd --timeout=180s
Configure ArgoCD HTTP mode
Required so the Cloudflare Tunnel can proxy to ArgoCD without TLS issues.
kubectl patch configmap argocd-cmd-params-cm \
  --namespace argocd --type merge \
  --patch '{"data":{"server.insecure":"true"}}'
kubectl rollout restart deployment argocd-server -n argocd
kubectl rollout status deployment argocd-server -n argocd
Get initial admin password
kubectl -n argocd get secret argocd-initial-admin-secret \
  -o jsonpath="{.data.password}" | base64 -d && echo
# Save this! You'll need it in Phase 5.
Install ArgoCD Image Updater
kubectl apply -n argocd \
  -f https://raw.githubusercontent.com/argoproj-labs/argocd-image-updater/stable/manifests/install.yaml

kubectl wait --for=condition=Ready pod \
  -l app.kubernetes.io/name=argocd-image-updater -n argocd --timeout=120s

# Set git commit identity for Image Updater commits
kubectl patch configmap argocd-image-updater-config \
  --namespace argocd --type merge \
  --patch '{"data":{"git.user":"ArgoCD Image Updater","git.email":"argocd-image-updater@schemavis.local"}}'
Install argocd CLI
curl -sSL -o argocd \
  https://github.com/argoproj/argo-cd/releases/latest/download/argocd-linux-amd64
chmod +x argocd && sudo mv argocd /usr/local/bin/
4
Cloudflare Tunnel + GitHub PAT
▼
Create Cloudflare tunnel
1. Go to one.dash.cloudflare.com → Networks → Tunnels → Create a tunnel
2. Name it schemavis → select Cloudflared connector → copy the tunnel token
3. Add Public Hostnames:
  • schemavis.shreyasshelar.uk → http://schema-vis-frontend.schema-vis.svc.cluster.local:80
  • argocd.shreyasshelar.uk → http://argocd-server.argocd.svc.cluster.local:80
4. Save tunnel
Create GitHub PAT (classic)
GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token
Scopes: repo (full), write:packages, read:packages
Copy the token (starts with ghp_...) — you can't see it again
Update values.yaml with imagePullSecrets
In helm/schema-vis/values.yaml, change both pullSecrets: [] lines to:
pullSecrets:
  - ghcr-credentials
Then commit and push to main.
5
Create k8s secrets + register repo
▼
Create namespace + generate JWT secret
kubectl create namespace schema-vis
JWT_SECRET=$(openssl rand -base64 64 | tr -d '\n')
echo "JWT_SECRET=$JWT_SECRET"   # save this!
Create backend secret
kubectl create secret generic schema-vis-backend-secret \
  --namespace schema-vis \
  --from-literal=SPRING_DATASOURCE_PASSWORD='YOUR_DB_PASS' \
  --from-literal=JWT_SECRET="$JWT_SECRET" \
  --from-literal=GEMINI_API_KEY='AIzaSy...' \
  --from-literal=GROQ_API_KEY='gsk_...' \
  --from-literal=OPENROUTER_API_KEY='sk-or-...'
Create Postgres + cloudflared secrets
kubectl create secret generic schema-vis-postgres-secret \
  --namespace schema-vis \
  --from-literal=POSTGRES_PASSWORD='YOUR_DB_PASS' \
  --from-literal=POSTGRES_USER='schemavis' \
  --from-literal=POSTGRES_DB='schemavis'

kubectl create secret generic schema-vis-cloudflared-secret \
  --namespace schema-vis \
  --from-literal=TUNNEL_TOKEN='eyJh...(your cloudflare tunnel token)...'
Create ghcr.io pull secrets
# In schema-vis namespace (pods pull images)
kubectl create secret docker-registry ghcr-credentials \
  --namespace schema-vis \
  --docker-server=ghcr.io \
  --docker-username=shreyasshelar \
  --docker-password='ghp_...(your GitHub PAT)...'

# In argocd namespace (Image Updater checks tags)
kubectl create secret docker-registry ghcr-credentials \
  --namespace argocd \
  --docker-server=ghcr.io \
  --docker-username=shreyasshelar \
  --docker-password='ghp_...(your GitHub PAT)...'
Register GitHub repo in ArgoCD
This gives Image Updater write access to commit image tag updates.
kubectl port-forward svc/argocd-server -n argocd 8080:80 &
argocd login localhost:8080 --username admin \
  --password 'YOUR_ARGOCD_PASSWORD' --insecure

argocd repo add https://github.com/shreyasshelar/SchemaVis.git \
  --username shreyasshelar \
  --password 'ghp_...(your GitHub PAT)...'
kill %1
6
Deploy & verify
▼
Clone repo and apply ArgoCD Application
git clone https://github.com/shreyasshelar/SchemaVis.git
cd SchemaVis
kubectl apply -f k8s/argocd-app.yaml
Watch pods come up
kubectl get pods -n schema-vis -w
# Expect: postgres → backend → frontend → cloudflared ×2
Verify backend health
kubectl exec -n schema-vis deploy/schema-vis-backend -- \
  wget -qO- http://localhost:8080/actuator/health
# Expected: {"status":"UP"}
Verify HPA created
kubectl get hpa -n schema-vis
# backend: 1–5 replicas  |  frontend: 1–3 replicas
Open the app in your browser
Open https://schemavis.shreyasshelar.uk — register an account and start designing your first schema!

🎉 Setup complete!

Your SchemaVis instance is live at schemavis.shreyasshelar.uk.
ArgoCD UI at argocd.shreyasshelar.uk.
Future deploys are fully automatic — just git push main.
