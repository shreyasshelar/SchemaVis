Deployment Flow

Press Play to animate the full journey — from a git push on your laptop to live pods serving traffic.
Total time: ~4–6 minutes first deploy, ~2–3 minutes with warm Docker layer cache.

1
📝
git push origin main
You push code to the main branch on GitHub. This is the only manual action in the entire pipeline. Branch protection ensures all CI checks pass before merge.
GitHub
2
⚙️
GitHub Actions CI — build & test
The ci.yml workflow triggers. Compiles Java 21 with Maven, runs the test suite against a real PostgreSQL service container, type-checks TypeScript with tsc --noEmit, and runs npm run build. Fails fast on any error.
GitHub Actions
3
🐳
Docker images built & pushed to ghcr.io
The deploy.yml workflow builds two multi-stage Docker images using BuildKit layer caching. Tags: sha-abc1234 (immutable, per-commit) and latest. Pushed to the private GitHub Container Registry (ghcr.io).
ghcr.io
4
🔍
ArgoCD Image Updater detects new tag
Image Updater polls ghcr.io every 2 minutes. It sees the new sha-abc1234 tag matching the ^sha- pattern. It writes the new tag to helm/schema-vis/values.yaml and commits: "chore: update backend to sha-abc1234" — full audit trail in git.
ArgoCD Image Updater
5
🔄
ArgoCD detects git change → diffs cluster
ArgoCD sees the new commit (image tag update). It computes a diff between desired state (git) and live cluster. Only the backend and frontend Deployments changed — PostgreSQL, cloudflared, Services, HPAs are untouched.
ArgoCD
6
🚀
Rolling update — zero downtime
k3s starts new pods with the new image before terminating old ones (RollingUpdate strategy). Spring Actuator /actuator/health/readiness must return 200 before any traffic shifts. HPA adjusts replicas based on CPU and memory load.
k3s
7
☁️
Cloudflare tunnel routes live traffic
The cloudflared pods (already running, never touched by this deploy) forward requests from schemavis.shreyasshelar.uk to the new frontend pods. PostgreSQL and cloudflared were never restarted — their state is preserved.
Cloudflare
8
✅
Deploy complete — full audit trail in git
Git history now contains a commit from ArgoCD Image Updater. Roll back at any time by reverting that commit — ArgoCD self-heals within seconds. No manual intervention, no SSH, no kubectl apply from your laptop.
GitOps

✓ Deployment Complete
