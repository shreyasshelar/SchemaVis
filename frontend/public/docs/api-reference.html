Messages (Chat)
POST /api/sessions/{id}/messages Send a message, get AI reply + diagram update 🔒 auth ▼
Request
Response
Notes
{ "content": "Add a payments table with Stripe integration" }
200 OK
{
  "messageId": "uuid",
  "content":   "Great! I've added a PAYMENTS table with...",
  "diagram":  "erDiagram\n  USERS { ... }\n  PAYMENTS { ... }\n",
  "complete": false
}

// When AI decides the schema is complete:
{ ..., "complete": true }

429 Too Many Requests  // Bucket4j rate limit hit (20 req/min/IP)
503 Service Unavailable // All AI providers unavailable

The AI response may contain a [DIAGRAM]...[/DIAGRAM] block with a Mermaid erDiagram.

If the AI includes [COMPLETE], the complete field is set to true and the schema is locked.

AI calls use a fallback chain: Gemini 2.5 Flash → Groq (Llama 3.3) → OpenRouter. The first available provider wins.

Response

{ "status": "UP" }

{ "status": "UP", "components": { "db": { "status": "UP" } } }
Error format
All error responses use a consistent shape 
{
  "code":      "AI_UNAVAILABLE",       // machine-readable
  "message":   "All AI providers are currently unavailable",
  "details":   [],                    // validation errors (optional)
  "timestamp": "2025-01-15T11:30:00Z"
}
